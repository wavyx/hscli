---
title: Configuration
description: Configure hscli with environment variables, profiles, and defaults.
---

hscli works with zero configuration after [login](/guides/authentication/). When you do
want to change defaults, you have two layers: **environment variables** (great for CI)
and **named profiles** (great for multiple accounts).

## Environment variables

| Variable | Description | Default |
| --- | --- | --- |
| `HSCLI_TOKEN` | Personal access token. Overrides the keychain when set. | — |
| `HSCLI_PROFILE` | Named profile to use. | `default` |
| `HSCLI_OUTPUT` | Default output format (`table`, `json`, `yaml`, `csv`). | `table` |
| `HSCLI_NO_COLOR` | Disable ANSI colors. | `false` |

```bash frame="terminal" title="~/.zshrc"
# make JSON the default everywhere
export HSCLI_OUTPUT=json
```

## Profiles

Each profile stores its own credentials in the keychain. Create one by logging in with
`--profile`, then target it per-command or via `HSCLI_PROFILE`.

```bash frame="terminal"
hscli auth login --profile staging
hscli conv list --profile staging
HSCLI_PROFILE=staging hscli report --from 2026-05-01
```

```bash frame="terminal" title="auth status"
$ hscli auth status
default   jordan@acme.io     ✓ valid
staging   bot@acme-test.io   ✓ valid
```

## Precedence

When the same setting is provided more than one way, hscli resolves it in this order
(highest first):

1. An explicit flag (`--output json`)
2. An environment variable (`HSCLI_OUTPUT`)
3. The built-in default

See [Output & filtering](/automation/output/) for everything you can do with output.
